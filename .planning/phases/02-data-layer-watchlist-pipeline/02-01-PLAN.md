<task type="auto" tdd="true">
  <name>Task 2: Implement universe ingestion and weekly watchlist scoring pipeline</name>
  <files>src/modules/data/universe.service.ts, src/modules/data/watchlist.service.ts, src/modules/data/pipeline.service.ts</files>
  <behavior>
    - Universe refresh reads canonical static source, validates symbols, and snapshots result.
    - If canonical source load fails, previous successful snapshot is reused.
    - Weekly selection scores eligible symbols with liquidity+trend+structure and selects target 5-10 (default 10).
    - Missing-input symbols are excluded and replaced by next-ranked candidates.
  </behavior>
  <action>Build deterministic scoring and ranking logic with explicit tie-break on liquidity. Persist component score breakdown and selected watchlist in one transaction per weekly run.</action>
  <verify>
    <automated>npm run test -- --runInBand universe watchlist scoring</automated>
  </verify>
  <done>Weekly pipeline deterministically produces persisted watchlist symbols from validated LQ45 universe.</done>
</task>

<task type="auto" tdd="true">
  <name>Task 3: Implement daily provider adapters and run wiring for market/indicator/sentiment ingest</name>
  <files>src/modules/data/providers/market-primary.client.ts, src/modules/data/providers/market-backup.client.ts, src/modules/data/providers/news.client.ts, src/modules/data/indicators.service.ts, src/modules/data/pipeline.service.ts, src/modules/queue/worker.ts, src/modules/runs/runs.service.ts</files>
  <behavior>
    - Daily run resolves current weekly watchlist and iterates each symbol.
    - OHLCV path uses primary provider then fallback provider on failure.
    - Indicator values are computed locally first and fallback to provider when local compute fails.
    - News sentiment is fetched and filtered per symbol keyword set.
    - If both market providers fail for a symbol/day, symbol is skipped and logged without failing entire run.
  </behavior>
  <action>Integrate phase-2 daily pipeline into existing run worker execution flow. Add retry budget, normalized adapter contracts, and symbol-level failure accounting.</action>
