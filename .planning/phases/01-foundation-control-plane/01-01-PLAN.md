<decision_traceability>
- D-01: Single Node control-plane process handles API + scheduler + worker.
- D-02: BullMQ worker runs in-process with concurrency 1.
- D-03: Docker Compose exposes one `node-control-plane` service container.
- D-04: Reliability baseline is `restart: unless-stopped` plus queue retry/backoff.
- D-06: Scheduled overlap produces `skipped_overlap` record.
- D-10: Model parameters persist inside provider profile records.
- D-11: Each run stores full effective AI config snapshot.
- D-16: Run records are retained without pruning in Phase 1.
</decision_traceability>
